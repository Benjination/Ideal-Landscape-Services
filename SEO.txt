═══════════════════════════════════════════════════════════
Last updated: March 2026
Status: Ready to implement once build option is confirmed.

This document is the implementation reference for every SEO feature in the
rebuild. It is written to be actionable — each item maps to a specific page,
component, or file and can be worked through section by section during the
build. Cross-referenced with thePlan.txt (design/structure) and structure.txt
or wordpress.txt (technical build checklist, whichever David chooses).

THE CORE PRINCIPLE
  Kathy's brief was right. Keyword stuffing is a 2009 technique. Google's
  current ranking signals reward:
    → Relevance (does the page clearly serve the search intent?)
    → Structure (can the crawler understand what the page is about?)
    → Authority (do other sites, reviews, GBP, and engagement confirm you?)
  
  The new site earns its rankings through architecture, metadata, image
  alt text, structured data, content organization, and a live review signal
  — not through repeated body copy. One well-written sentence per section
  beats three paragraphs of padded keyword repetition.

6. LIVE REVIEW INTEGRATION
════════════════════════════════════════════════════════
  Static hardcoded testimonials go stale and carry no live SEO signal.
  A live Google review feed does both — shows visitors real, recent reviews
  AND signals to Google's local algorithm that the business has active review
  engagement.

  RECOMMENDED INTEGRATION OPTIONS (in order of preference)
    1. EmbedSocial (embedsocial.com) — $29–$49/mo
       Clean widget, strong design control, pulls Google + Facebook reviews.
    2. Widewail (widewail.com) — $30–$60/mo
       Full review management + embed. Also helps solicit new reviews.
    3. Elfsight (elfsight.com) — $9/mo basic tier, free tier available
       Very easy WordPress embed, basic display options.
    4. WP Business Reviews (WordPress plugin) — $49/yr
       Self-contained WordPress solution, no monthly fee.
    5. Free embed fallback: Google Maps Place embed + manual review widget.
       Less dynamic but zero cost.
  
  Custom build alternative: Google Places API directly ($0 for <28,500 calls/mo).
    → Fetches ratings and review count in real time.
    → Pull into a static-generation step in Next.js (ISR or build-time fetch)
       so it renders as plain HTML — no client-side API calls needed.

  PLACEMENT ON SITE
    - Homepage "Happy Clients" section: show 3–5 featured reviews dynamically.
    - Trust signal strip: live star rating + review count badge
      ("⭐ 4.9 · 47 Google Reviews" updates automatically).
    - Testimonials page: full review feed.

  EXISTING TESTIMONIALS (from site-content/testimonials.txt)
    The four existing testimonial letters (M. Madden, J. Clifford, C. Langford,
    M. & S. Davis) are long, genuine, and very specific. These are worth keeping

8. SCHEMA MARKUP (structured data Google reads directly)
════════════════════════════════════════════════════════
  Schema is embedded in the HTML (typically as JSON-LD in a <script> tag).
  It does not change the visual appearance of the page but can produce rich
  search results (star ratings, address, FAQ dropdowns, etc.) in Google.

  A. LocalBusiness Schema (homepage — highest priority)
  ─────────────────────────────────────────────────────
  Place in the <head> of the homepage. Template:
  {
    "@context": "https://schema.org",
    "@type": "LandscapeDesignService",    ← or "HomeAndConstructionBusiness"
    "name": "Ideal Landscape Services",
    "telephone": "817-457-7507",
    "url": "https://ideallandscapeservices.com",
    "address": {
      "@type": "PostalAddress",
      "addressLocality": "[City — confirm with David]",
      "addressRegion": "TX",
      "addressCountry": "US"
    },
    "areaServed": [
      "Fort Worth", "Dallas", "Allen", "Plano", "Frisco",
      "McKinney", "Arlington", "Southlake", "Colleyville", "Keller"
    ],
    "sameAs": [
      "https://www.facebook.com/IdealLandscapeServices",
      "https://www.instagram.com/ideallandscapeservices/"
    ],
    "image": "[URL of best logo or hero image on new site]",
    "priceRange": "$$"
  }
  Note: Use LandscapeDesignService if available; fallback is
  HomeAndConstructionBusiness or LocalBusiness.

  B. FAQ Schema (FAQ page — enables FAQ rich results in Google)
  ─────────────────────────────────────────────────────────────
  All 11 FAQs (archived in site-content/faq.txt) are perfect candidates.
  FAQ schema can produce expandable Q&A dropdowns directly in Google search
  results — no click needed. High visibility, zero extra effort once written.
  Template per question:
  {
    "@type": "Question",
    "name": "What's a French drain?",
    "acceptedAnswer": {
      "@type": "Answer",
      "text": "A French drain is often added to property where the natural
               slope isn't enough to move excess storm water off the property..."
    }
  }
  Implement as a full FAQPage JSON-LD on /faq/. Include all 11 questions.

  C. BreadcrumbList Schema (service pages and blog posts)
  ───────────────────────────────────────────────────────
  Adds breadcrumb trail to Google search snippet:
    Ideal Landscape Services > Services > Water Features
  Helps clarify site hierarchy in search results. Auto-generated by Yoast
  in WordPress; manual JSON-LD in custom build.

  D. BlogPosting Schema (each blog post)
  ───────────────────────────────────────
  Tells Google the post's author, date published, date modified, and topic.
  In WordPress: Yoast SEO handles this automatically.
  In custom build: add to each blog post's page component.

  E. Review / AggregateRating Schema (homepage, once review integration active)
  ─────────────────────────────────────────────────────────────────────────────
  If using Google Places API directly, can embed the aggregate star rating in
  the schema — enables star ratings to appear in organic search results for
  the business name. High CTR boost.

9. TECHNICAL SEO CHECKLIST
════════════════════════════════════════════════════════

  PRE-LAUNCH (must be in place before going live)
  ─────────────────────────────────────────────────
  [ ] XML Sitemap
      Auto-generated. Include all public pages; exclude /admin/, /wp-admin/,
      thank-you pages. Submit to Google Search Console at launch.
      WP: Yoast generates automatically. Custom: next-sitemap package.

  [ ] robots.txt
      Allow: all content pages, all image folders.
      Disallow: /admin/, /wp-admin/, /wp-login.php, ?s=* (search queries),
                /cart/, /checkout/ (if WooCommerce ever added).
      Custom build robots.txt should be minimal — most disallows are WP-specific.

  [ ] Canonical URLs
      Every page must declare its canonical URL in <head>:
        <link rel="canonical" href="https://ideallandscapeservices.com/services/water-features/" />
      Prevents duplicate content issues from URL parameters, trailing slashes, etc.
      WP: Yoast handles automatically. Custom: add to page <Head> component.

  [ ] 301 REDIRECT MAP (critical — preserves link equity from old site)
      The 8 blog posts on the current site have their own URLs that may have
      inbound links or accumulated search position. These MUST redirect to the
      new blog post URLs or all that equity evaporates.
      
      Map (old URL → new URL — confirm new slugs at build time):
        /mulch-too-mulch/                                → /blog/mulch-too-mulch/
        /what-is-scalping-anyway/                        → /blog/what-is-scalping/

        /products-and-services/water-features/           → /services/water-features/
        /products-and-services/landscape-design-ideas/   → /services/landscape-design/
        /products-and-services/landscape-maintenance/    → /services/maintenance/

        /contact-us/                                     → /contact/

      Custom build: _redirects file (Firebase / Netlify) or next.config.js redirects.

  [ ] HTTPS
      Firebase Hosting: auto-provisions SSL. No action needed.
      SiteGround / Bluehost: Let's Encrypt SSL, usually one-click in cPanel.
      Enforce HTTPS redirect (HTTP → HTTPS) at the server/hosting level.

  [ ] Open Graph + Twitter Card Meta Tags
      Controls how the site looks when shared on Facebook, Instagram, LinkedIn, iMessage.
      Required tags per page:
        og:title, og:description, og:image, og:url, og:type
        twitter:card, twitter:title, twitter:description, twitter:image
      OG image: 1200×630px. Use a high-quality landscape photo as default
      (water feature or outdoor living shot). Per-page OG images are ideal
      for blog posts (use the post's featured image).
      WP: Yoast SEO handles all of these.
      Custom: Add to page <Head> component, pass as props from page data.

  [ ] Image Optimization (page speed + Core Web Vitals)
      · Serve all images in WebP format (30–50% smaller than JPEG/PNG, same quality).
        Next.js <Image> component converts and serves WebP automatically.
        WP: Smush or ShortPixel plugin handles WebP conversion on upload.
      · Lazy-load all images below the fold (native loading="lazy" attribute).
        Next.js: default behavior. WP Elementor: enable lazy load in settings.
      · Hero image MUST NOT be lazy-loaded (it is the LCP element).
        Next.js: priority={true} on the hero <Image>.
        WP Elementor: exclude hero from lazy load.
      · Largest Contentful Paint (LCP) target: under 2.5 seconds.
        Achieve by: serving hero from CDN (Firebase Hosting has global CDN
        built in; SiteGround has CloudFlare), preloading the hero image URL,
        keeping the hero image file under ~300kb in WebP at display size.

  [ ] Page Speed — General
      Custom build: Next.js is fast by default — ISR, code splitting, tree shaking.
        · Avoid loading large JS libraries client-side (carousel, maps, reviews)
          unless they are code-split and lazy-loaded on interaction.
        · Google Maps embed: use a static map image with a click-to-load overlay
          (fastest pattern — actual Google Maps iframe only loads on click).
      WP build:
        · Enable SiteGround's SuperCacher or built-in WP caching.
